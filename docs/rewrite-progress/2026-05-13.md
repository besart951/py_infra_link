## Why This Candidate

Six classes of inconsistency were identified during the audit that directly
harm API clients, violate clean-architecture Seam discipline, and introduce
hidden complexity in each Module's Interface:

1. **Wrong HTTP status codes** — ConflictError variants returned 400 instead of
   409; ValidationError variants returned 400 instead of 422.  A client cannot
   distinguish "name already taken" from "name is too short" without reading the
   error message text.

2. **Error-mapping parameter type** — Nine mappers accepted `Exception` instead
   of `DomainError`, hiding the domain-level contract from the type-checker.

3. **User module schema divergence** — `UserResponse` lacked `from_attributes`,
   `id` was typed as `str` instead of `UUID`, and pagination used a bespoke
   `UserPageResponse` class instead of the shared `Page[T]`.  The resulting
   `_to_response()` helper was shallow pass-through code that added complexity
   without providing any leverage.

4. **Module factory duplication** — Seven route files recreated the Module
   instance inside every single handler (4–5 times per file, 35 total).
   Changing a constructor argument required touching every handler.

5. **UUID import style** — `notification/schemas.py` used `uuid.UUID` where all
   other schemas import `UUID` directly.

6. **Duplicate 404 branch** — `project_resource_link/error_mapping.py` checked
   `BuildingDoesNotExistError` in a separate `if` block from the other 404
   errors, creating dead-code noise.

## Seam Decision

No Seam changes.  Error mappers are presentation-layer adapters; the Seam

## Interface Chosen

The `error_mapping` functions now expose a narrower Interface:
`(error: DomainError) -> HTTPException`.  Previously the parameter type was
`Exception`, which was broader than necessary and bypassed pyright's exhaustive
error-variant checks.

### User module — schemas and routes

- `UserResponse`: added `from_attributes=True`, `frozen=True`; changed `id:
  str` → `id: UUID`; added `UUID` import.
- Deleted `UserPageResponse` (shallow wrapper around `Page[T]` with computed
  fields that `Page` already provides as properties).
- `user/routes.py`: deleted `_to_response()` helper; added `_make_module()`
  factory; replaced `UserPageResponse` with `Page[UserResponse]`; removed the
  unused `datetime` and `PageParams` imports.

### Module factory — 7 route files

Added `_make_module(session)` factory to facility, building, control\_cabinet,
sps\_controller, sps\_controller\_system\_type, field\_device, bacnet\_object.
Each handler now calls `_make_module(session)` instead of constructing the
Module inline.  (project and notification already used this pattern.)

### Notification schemas

Changed `id: uuid.UUID` / `user_id: uuid.UUID` → `id: UUID` / `user_id: UUID`
with a direct `from uuid import UUID` import, matching all other schemas.
