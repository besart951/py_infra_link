# 2026-05-13 Session: Completing the Integration Test Suite

## Summary

Expanded the integration test suite to cover all 12 SQLAlchemy repository adapters
and all 7 remaining physical-hierarchy adapters. Added `pytest.mark.integration` marker
for clean CI separation. 218 tests pass (145 unit + 73 integration).

## Work Done

### 1. pytest.mark.integration marker

- Added `markers` entry to `[tool.pytest.ini_options]` in `pyproject.toml`.
- Added `pytestmark = pytest.mark.integration` module-level variable to all 5 existing
  integration test files.
- This allows developers to run `pytest -m "not integration"` for a fast (0.5s) unit
  feedback loop, and `pytest -m integration` for the PostgreSQL-backed suite.

### 2. CI workflow: separate unit + integration steps

Updated `.github/workflows/ci.yml` to replace the single `Tests (pytest)` step with:

```yaml
- name: Unit tests (pytest)
  run: uv run pytest -m "not integration" --tb=short -q

- name: Integration tests (pytest + testcontainers)
  run: uv run pytest -m integration --tb=short -q
```

GitHub Actions ubuntu-latest ships with Docker, so testcontainers works without a
dedicated `services:` block. The split makes CI failures easier to diagnose (unit
failures are fast and appear before the slower integration step).
