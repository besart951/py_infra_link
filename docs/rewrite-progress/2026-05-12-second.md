---

## Interface Design

**Design It Twice** applied internally:

1. **Minimal Interface** (2 entry points: upsert + list) — rejected: cannot express partial updates cleanly; forces callers to read-then-write.
2. **Common Caller Interface** (5 entry points: create, get, list, update, delete) — **chosen**. Follows the established domain pattern, expresses intent clearly at each call site.
3. **Ports & Adapters** — not justified; only one real persistence adapter is needed.

The Interface enforces:
- `BacnetObjectName.parse()` strips whitespace and validates length (≤ 100 chars).
- `BacnetObjectInstance.parse()` validates non-negative integer.
- `(device_id, object_type, object_instance)` uniqueness — the BACnet addressing constraint.
- `(device_id, name)` uniqueness — BACnet requires unique object names per device.
- Parent Field Device must exist before creation.
