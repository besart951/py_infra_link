## Interface Design

**Design It Twice** applied:

1. **Minimal Interface** (2 entry points: upsert + list) — rejected: cannot express partial update cleanly; forces callers into read-then-write.
2. **Common Caller Interface** (5 entry points: create, get, list, update, delete) — **chosen**. Follows the established domain pattern, expresses intent, scopes all operations by `owner_id` to enforce ownership.
3. **Ports & Adapters with ownership port** — not justified; ownership is simple ID comparison, not a separate Seam.

The Interface enforces:
- `ProjectName.parse()` strips whitespace and validates length (≤ 100 chars).
- `(owner_id, name)` uniqueness — a User cannot have two Projects with the same name.
- Owner User must exist before creation.
- `get`, `update`, `delete` all check `owner_id` match — cross-owner access returns `ProjectNotFoundError` (no information leak).
